# Стейкхолдер
Команда разработки — проверяет работоспособность интеграции с внешним API перед тем, как расширять объём загружаемых данных.

# Проблема
Прямой скраппинг HLTV заблокирован анти-бот защитой (задачи 002, 003 на паузе). Перед тем как строить аналитические компоненты (007–009), нужно убедиться, что альтернативный источник данных вообще работает: API доступен, данные приходят в ожидаемом формате, схема БД корректна.

# Желаемый результат
1. Данные о командах (с рейтингами) и их матчах из внешнего API-источника появляются в базе данных.
2. Начальная загрузка охватывает историю матчей за последнюю неделю для команд из топ-10 рейтинга CS2.
3. Загрузку данных можно запустить вручную и получить актуальный срез матчей в базе данных.

# Контекст
PandaScore предоставляет официальный REST API для CS2 с бесплатным планом (1 000 запросов/час). Это исключает необходимость в Selenium, обходе анти-бот защиты и сложной batch-инфраструктуре.

Триггер задачи: задачи 002 и 003 (HLTV-скраппинг) поставлены на паузу из-за сложности обхода защиты. PandaScore выбран как альтернативный источник — проба пера: проверить интеграцию на минимальном объёме данных, прежде чем расширять горизонт загрузки.
