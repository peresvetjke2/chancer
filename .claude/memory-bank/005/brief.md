# Стейкхолдер
Пользователь-бетор, которому нужны прогнозы на матчи. Задачи 007–009
(аналитика формы, H2H, прогноз) — прямые потребители этих данных.

# Проблема
Текущая интеграция с PandaScore (задача 004) сохраняет только минимальный
набор полей. API возвращает значительно больше структурированных данных,
которые нужны для аналитики формы команд, H2H и генерации прогнозов,
но они молча отбрасываются при импорте.

Конкретные разрывы:

1. **Результаты карт не сохраняются.** При импорте матчей данные по картам
   из поля `games[]` API не сохраняются. Для H2H и анализа формы нужны счёт
   и победитель по каждой карте, а не только итоговый счёт серии.

   ```json
   "games": [
     { "map": { "name": "Mirage" },  "winner": { "id": 129407 }, "results": [{"team_id": 129407, "score": 16}, {"team_id": 98765, "score": 10}] },
     { "map": { "name": "Inferno" }, "winner": { "id": 98765  }, "results": [{"team_id": 129407, "score": 13}, {"team_id": 98765, "score": 16}] },
     { "map": { "name": "Ancient" }, "winner": { "id": 129407 }, "results": [{"team_id": 129407, "score": 16}, {"team_id": 98765, "score": 12}] }
   ]
   ```

2. **Детали матча обрезаны.** Отсутствуют: `begin_at` (время начала),
   `match_type` (bo1/bo3/bo5), `league.name`, `serie.name`, `status`.
   Без `match_type` счёт 2:0 неоднозначен: в bo3 это победа 2–0, в bo5 —
   проигравший ещё в игре; задача 008 (H2H) не может их различить.
   Без `begin_at` невозможно строить таймлайн формы команды.

   ```json
   "begin_at":   "2024-03-15T14:00:00Z",
   "end_at":     "2024-03-15T16:30:00Z",
   "match_type": "best_of_3",
   "status":     "finished",
   "league":     { "id": 456, "name": "ESL Pro League" },
   "serie":      { "id": 789, "name": "Season 19" }
   ```

3. **Команды без идентификаторов.** Не сохраняются `acronym`, `image_url`,
   `slug`. Нужны для отображения и дедупликации.

   ```json
   "acronym":   "NaVi",
   "slug":      "natus-vincere",
   "image_url": "https://cdn.pandascore.co/images/team/image/129407/natus_vincere.png"
   ```

4. **Ранг команды — псевдоданные.** `pandascore_rank` назначается по позиции
   в массиве ответа, а не из реального поля рейтинга. Значение ненадёжно.

   ```json
   // реальное поле в ответе:
   "ranking": 3
   ```

5. **Игроки без надёжной идентификации.** В таблице `players` нет
   `pandascore_id`; механизм upsert по нему отсутствует. При повторном
   импорте каждый игрок создаётся заново — дубли накапливаются.

   ```json
   // players[] инлайн в ответе /csgo/teams:
   "players": [
     { "id": 11234, "name": "s1mple", "role": "rifler" },
     { "id": 11235, "name": "b1t",    "role": "rifler" }
   ]
   ```

# Желаемый результат
1. Задача 008 (H2H) и 007 (форма) могут анализировать результат каждой
   карты в матче: название карты, счёт и победителя — без дополнительных
   запросов к API.
2. Задача 008 (H2H) может фильтровать матчи по типу (bo1/bo3/bo5)
   и различать счёт 2:0 в bo3 от 2:0 в bo5. Задача 007 (форма) может
   строить таймлайн с точностью до времени начала матча. Матчи можно
   группировать по лиге, сезону и турниру.
3. Задача 007 (форма) может отображать логотип и аббревиатуру команды
   без дополнительных запросов к API. Ранг команды отражает реальный
   рейтинг PandaScore, а не порядок в ответе.
4. При каждом запуске импорта один и тот же игрок из PandaScore всегда
   соответствует одной и той же записи в таблице players — без дублей.
   > **Ограничение scope:** трансферы игроков (изменение `team_id`, `role`)
   > при повторном импорте не обрабатываются. Это отдельная задача.

# Контекст
Задачи 007 (форма команды), 008 (H2H), 009 (прогноз) напрямую зависят
от карточной статистики и типа матча. Расширить сохранение данных сейчас
дешевле, чем переимпортировать историю позже.
