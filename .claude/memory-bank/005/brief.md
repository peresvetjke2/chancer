# Стейкхолдер
Команда разработки — расширяет объём данных PandaScore для поддержки
аналитических задач (007–009).

# Проблема
Текущая интеграция с PandaScore (задача 004) сохраняет только минимальный
набор полей. API возвращает значительно больше структурированных данных,
которые нужны для аналитики формы команд, H2H и генерации прогнозов,
но они молча отбрасываются при импорте.

Конкретные разрывы:

1. **Результаты карт не сохраняются.** Таблица `map_results` существует, но
   `MatchesImporter` игнорирует массив `games[]` в ответе. Для H2H и анализа
   формы нужны счёт и победитель по каждой карте, а не только итоговый счёт
   серии.

   ```json
   "games": [
     { "map": { "name": "Mirage" },  "winner": { "id": 129407 }, "results": [{"team_id": 129407, "score": 16}, {"team_id": 98765, "score": 10}] },
     { "map": { "name": "Inferno" }, "winner": { "id": 98765  }, "results": [{"team_id": 129407, "score": 13}, {"team_id": 98765, "score": 16}] },
     { "map": { "name": "Ancient" }, "winner": { "id": 129407 }, "results": [{"team_id": 129407, "score": 16}, {"team_id": 98765, "score": 12}] }
   ]
   ```

2. **Детали матча обрезаны.** Отсутствуют: `begin_at` (время начала),
   `match_type` (bo1/bo3/bo5), `league.name`, `serie.name`, `status`.
   Без `match_type` нельзя корректно интерпретировать счёт; без `begin_at`
   нельзя строить таймлайн.

   ```json
   "begin_at":   "2024-03-15T14:00:00Z",
   "end_at":     "2024-03-15T16:30:00Z",
   "match_type": "best_of_3",
   "status":     "finished",
   "league":     { "id": 456, "name": "ESL Pro League" },
   "serie":      { "id": 789, "name": "Season 19" }
   ```

3. **Команды без идентификаторов.** Не сохраняются `acronym`, `image_url`,
   `slug`. Нужны для отображения и дедупликации.

   ```json
   "acronym":   "NaVi",
   "slug":      "natus-vincere",
   "image_url": "https://cdn.pandascore.co/images/team/image/129407/natus_vincere.png"
   ```

4. **Ранг команды — псевдоданные.** `pandascore_rank` назначается по позиции
   в массиве ответа, а не из реального поля рейтинга. Значение ненадёжно.

   ```json
   // реальное поле в ответе:
   "ranking": 3
   ```

5. **Игроки без PandaScore ID.** Колонки `pandascore_id` нет в таблице
   `players`, что блокирует задачу 005 (статистика игроков).

   ```json
   // players[] инлайн в ответе /csgo/teams:
   "players": [
     { "id": 11234, "name": "s1mple", "role": "rifler" },
     { "id": 11235, "name": "b1t",    "role": "rifler" }
   ]
   ```

# Желаемый результат
1. Результаты карт (карта, счёт, победитель) сохраняются для каждого матча.
2. Матчи хранят `begin_at`, `match_type`, `league`, `serie`, `status`.
3. Команды хранят `acronym`, `image_url`, `slug`; ранг берётся из реального
   поля API.
4. Таблица `players` получает `pandascore_id` — задел для задачи 005.

# Контекст
Задачи 007 (форма команды), 008 (H2H), 009 (прогноз) напрямую зависят
от карточной статистики и типа матча. Расширить сохранение данных сейчас
дешевле, чем переимпортировать историю позже.
