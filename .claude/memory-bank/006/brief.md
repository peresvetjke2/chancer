# Стейкхолдер
Аналитическая система (задачи 007, 008, 009) — не может строить статистически значимые
прогнозы без достаточного объёма исторических матчей.

# Проблема
Задача 004 заложила интеграцию с PandaScore на минимальном объёме. Перед тем как строить аналитику, нужно наполнить БД репрезентативной историей матчей — за последние 3 месяца и по расширенному кругу команд.

Конкретные разрывы:

1. **Нет механизма задать период сбора.** Текущий импорт не параметризован по датам.
   Нельзя сказать «загрузи матчи с 01.01 по 01.04» — это нужно для первоначального
   наполнения и повторного запуска при расширении горизонта.

2. **Неизвестен нужный круг команд.** Вместо того чтобы заранее задавать список вручную,
   правильнее взять все матчи за период и получить из них реальный состав участников.
   Это позволяет базе данных отражать то, кто фактически играл в турнирах, а не
   субъективный топ-N.

3. **Пагинация не реализована.** Эндпоинт `/csgo/matches/past` возвращает максимум 100
   матчей за запрос. За 3 месяца матчей значительно больше — без пагинации данные неполные.

# Желаемый результат
1. В БД загружены все CS2-матчи за последние 3 месяца, без ограничения в 100 записей.
2. По итогу импорта известен полный список `pandascore_id` команд-участников —
   как входные данные для задачи 013 (сбор деталей команд).
3. Повторный запуск не создаёт дублей и не теряет данные.

# Контекст
## Эндпоинт
`GET /csgo/matches/past` ([документация](https://developers.pandascore.co/reference/get_csgo_matches_past))

Подтверждённые параметры фильтрации:
- `filter[begin_at]=YYYY-MM-DD` — фильтр по дате начала (время игнорируется, только
  день/месяц/год)
- `range[begin_at]=YYYY-MM-DD,YYYY-MM-DD` — диапазон дат; синтаксис `range[]`
  подтверждён документацией как механизм для интервалов
- `page[size]` (1–100, по умолчанию 50), `page[number]` — пагинация
- `filter[team_id]=123,456` — необязательный фильтр по командам; в данной задаче
  не используется (берём все матчи за период)

## Риски
- **`range[begin_at]` не проверен на реальных данных.** Документация подтверждает
  механизм `range[]` в целом, но конкретные примеры для полей типа datetime отсутствуют.
  При несоответствии ожидаемому поведению потребуется альтернативный способ фильтрации
  по дате (например, постраничный перебор с ручной фильтрацией по `begin_at` в ответе).

## Зависимости
- **Задача 005** — схема `match_games` уже существует; матчи сохраняются через
  существующий механизм дедупликации по `pandascore_id`.
- **Задача 013** — получает список команд-участников как выход этой задачи.
- **Задачи 007, 008, 009** — потребители данных; требуют не менее 3 месяцев истории
  для статистически значимой аналитики.

## Ограничения
- Бесплатный план PandaScore: 1 000 запросов/час. При 100 матчах на страницу
  и ~500 матчах в месяц потребуется около 15 запросов на 3 месяца сбора матчей.
  Это укладывается в лимит с большим запасом.
- Scope задачи — только матчи (`matches` + `match_games`). Детали команд-участников —
  задача 013.
