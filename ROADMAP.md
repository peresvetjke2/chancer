## Архитектурные решения

### Источник данных: PandaScore вместо HLTV

**Решение (2026-04-04):** PandaScore предоставляет структурированный REST API для CS2. Это исключает необходимость в Selenium, обходе анти-бот защиты и batch-инфраструктуре.

Ключевые эндпоинты:

| Эндпоинт | Назначение |
|----------|------------|
| `GET /csgo/teams` | Список команд с рейтингом |
| `GET /csgo/matches/past` | История матчей (фильтр по команде, дате) |
| `GET /csgo/matches/upcoming` | Предстоящие матчи |
| `GET /csgo/matches/{id}` | Детали матча + карты |
| `GET /csgo/players/{id}/stats` | Статистика игрока |
| `GET /tournaments/{id}/rosters` | Составы команд |

Все запросы с `filter[videogame_title]=cs-2`.

### Схема БД: добавление PandaScore ID

**Решение (2026-04-04):** Не переименовывать существующие `hltv_id`-поля, а добавить `pandascore_id` как новый столбец там, где нужно.

Изменения схемы в рамках 004:

| Таблица | Добавить | Примечание |
|---------|----------|------------|
| `teams` | `pandascore_id` (integer, unique) | Рядом с существующим `hltv_id` |
| `teams` | `pandascore_rank` (integer) | Рядом с `hltv_rank` |
| `matches` | `pandascore_id` (integer, unique) | Ключ дедупликации |
| `players` | `pandascore_id` (integer, unique) | Задача 005 |

Стратегия дедупликации: upsert по `pandascore_id`. Команды-оппоненты, отсутствующие в БД, создаются как минимальные записи (`pandascore_id` + `name`).

### Batch-сбор матчей и команд (задачи 012–013)

**Задача 012 — Batch-сбор прошлых матчей за период.**

Эндпоинт: `GET /csgo/matches/past` ([docs](https://developers.pandascore.co/reference/get_csgo_matches_past)).

Параметры фильтрации:
- По команде: `filter[team_id]=<id>` (id, не slug; поддерживаются несколько через запятую)
- По дате (точное значение): `filter[begin_at]=YYYY-MM-DD` — время игнорируется, только дата
- По диапазону дат: параметр `range[]` (конкретный синтаксис для поля `begin_at` — см. открытый вопрос ниже)

Сценарий использования: задаём период (например, последний месяц) → получаем все матчи → из них извлекаем список команд для задачи 013.

**Задача 013 — Сбор данных о командах по ID/slug.**

Эндпоинт: `GET /csgo/teams/{teamIdOrSlug}` ([docs](https://developers.pandascore.co/reference)).

Параметр пути `{teamIdOrSlug}` принимает **и числовой id, и строковый slug** — оба варианта подтверждены документацией.

Сценарий использования: из матчей задачи 012 собираем список команд → для команд, отсутствующих в БД или требующих обновления, делаем запрос по id → upsert по `pandascore_id`.

### Детальная статистика игроков (задача 005)

Статистика по картам (kills/deaths/rating) требует отдельного запроса на каждый матч — `GET /csgo/matches/{id}/games/{game_id}`. Начинаем без неё: сначала матчи + счёт по картам, player_stats — отдельная задача 005.
